# AI-Assisted Coding Checklist

## Before Coding

- [ ] Task is scoped to a clear outcome.
- [ ] Relevant files have been inspected.
- [ ] Data privacy and security constraints are known.
- [ ] Acceptance criteria are written.
- [ ] Test strategy is clear.

## During Coding

- [ ] AI-generated changes are reviewed before acceptance.
- [ ] New abstractions are justified.
- [ ] No unrelated refactors are included.
- [ ] Secrets, tokens, and private data are not committed.
- [ ] Generated code follows repository conventions.

## Before Merge

- [ ] Unit tests pass.
- [ ] Lint checks pass.
- [ ] Documentation is updated where needed.
- [ ] Known limitations are documented.
- [ ] Reviewer can reproduce the result locally.
